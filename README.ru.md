# My Raid CDs

[English](README.md) | **Русский**

Веб-приложение для учёта рейдовых КД по персонажам и подземельям (фокус на WotLK). Данные хранятся локально в `localStorage`.

## Возможности

Панели тулбара (подбор персонажа, подбор софтов, BIS-сборки, добавление персонажа/подземелья, данные) взаимоисключающие и используют одну оболочку. Формы персонажа/подземелья узкие (480px); блок **Данные** — сетка 2×2 (680px); BIS — до 1280px; **Подбор персонажа** и **Подбор софтов** — до 1920px, чтобы unit-grid не растягивался на ультрашироких экранах.

### Персонажи и подземелья

Добавление вручную или загрузка шаблона рейдов WotLK, когда список пуст. В форме нового подземелья есть подсказки названий рейдов и автозаполнение коротких аббревиатур. Редактирование имени, спеков, экипировки (импорт WowSims), метаданных подземелья и эмблем. Диалог правки напоминает переимпортировать гир или обновить GS, если поля разъехались.

### Переключатели КД

Переключатели персонаж–подземелье; сброс по персонажу из заголовка таблицы или сброс всех из панели **Данные**.

### Управление данными

Тулбар **Данные** (RU) / **Data** (EN). Блоки unit-размера: сброс всех КД, удаление всех персонажей, удаление всех подземелий (или **Добавить из шаблона**, если список пуст), удаление всех локальных BIS-списков. Деструктивные действия с подтверждением.

### Таблица

Сортировка по имени, типу (размер + героик), ilvl или числу прохождений; смешанный поиск по названию рейда, размеру и режиму (например `ICC`, `Uld10`, `ICC25N` / `ЦЛК25об`, `ToC25H` / `ИК25хм`; EN/RU имена работают в любой локали); компактный вид на узких экранах.

### Подбор персонажа

Копирование состава персонажей без КД по видимым (отфильтрованным) рейдам. Тулбар **Подбор персонажа** (RU) / **Character pick** (EN). Сетка фильтров равной ширины: **мин. GS**, **роль** (иконки LFG 2×2), **спеки** (2×1; **Выбрать все** / **Снять все**), **рейды** (поиск таблицы, чипы с иконками). **Сбросить все фильтры** в шапке панели. Чекбоксы спеков синхронизируются с поиском рейдов: персонажи на КД по всем видимым рейдам сбрасываются и отключаются. Неактивные строки — курсив/приглушение/ч/б или зачёркивание спека при блокировке ролью/GS; подсказки по наведению объясняют почему. При ширине ≥1600px строки результата справа от фильтров. У каждой строки рейда есть **Копировать**.

### Подбор софтов

Планирование софт-резервов для одного персонажа + спека. Тулбар **Подбор софтов** (RU) / **Soft pick** (EN). Правила рейда (макс. софтов 1–4; **реролл** или **+100**), выбор main/off спека, свои софты и чужие заявки (гистограмма) по BiS / BiS-вариантам из отфильтрованных рейдов. Для ICC и ИК боссы идут в порядке энкаунтеров; вставка — строки вида `- Item (Boss) xN `. Вёрстка: столбик на узких экранах; на средних софты рядом с фильтрами, блок вызова ниже; на широких (≥1600px) вызов рядом с софтами. Только на сессию (не сохраняется).

### BIS-сборки

Встроенные пресеты по спекам (Titans + community); тулбар **BIS сборки** (RU) / **BiS builds** (EN). Локальные копии редактируются; ими питаются подсказки апгрейдов. Слоты — paper-doll (парная таблица брони слева/справа + оружие снизу) с иконками пустых слотов WoW (плащ = иконка нагрудника; рубашка/гербовая накидка — только косметические заглушки). Встроенные списки только для чтения; усечённые имена слотов в компактном виде. Ошибки сохранения (квота и т.п.) — алерт; битые локальные записи при загрузке пропускаются.

### Подсказки гира

Янтарный = недостающие цели выбранного BiS; синий = апгрейды по ilvl со стат-фильтром (темнее = больше слотов). Dual-spec делит ячейку: слева main, справа off. Кольца и тринкеты считаются закрытыми в любой слот пары. Обычные/гер. варианты с одним именем — один предмет для BiS и ilvl. Фракционные близнецы одного ilvl (например Solace Альянс/Орда) закрывают BiS, но другой id может остаться на ilvl-треке. Ilvl-подсказки фильтруют прок-тринкеты по роли. Рубиновое святилище фильтрует лут по размеру и сложности. Легенду над таблицей можно скрыть. В тултипах секции **BiS** / **Upgrades**, без лута чужого режима; токены тира (в т.ч. Склеп Аркавона); боссы ICC/ИК в порядке энкаунтеров; в светлой теме — тёмная контрастная панель тултипа. Для танков на ilvl нужны defense/dodge/parry (BiS перекрывает).

### EN / RU

Полный UI + тултипы предметов (Cavern of Time / WoWRoad). При первом визите (нет сохранённой локали) по умолчанию **русский**.

### Тема

Светлая/тёмная тема со slate-оформлением (матовый хедер, карточки таблицы и панелей), сохраняется локально. Герб в хедере (`logo.svg`); favicon — `logo-as-icon.svg`. В хедере также ссылки на GitHub и [sergimax.ru](https://sergimax.ru) (иконка дома).

## Разработка

**Стек:** React 19, TypeScript, Vite, MUI, Vitest + Testing Library.

**CI:** На push/PR в `main` GitHub Actions параллельно гоняет **Lint**, **Test** и **Build** (`.github/workflows/ci.yml`); push в `main` также загружает артефакт `dist` (`.github/workflows/build-artifacts.yml`).

**Структура:** `src/components/` (UI), `src/hooks/` (домен + оверлей-панели), `src/utils/`, `src/data/` (бандлы WoW + BiS-пресеты), `src/storage/`. Тесты рядом: `*.test.ts(x)`.

Соглашения для контрибьюторов/агентов: [`.cursor/rules/project-rules.mdc`](.cursor/rules/project-rules.mdc).

**Роадмап:** [docs/roadmap.md](docs/roadmap.md).

### Быстрый старт

```bash
npm install
npm run dev
```

Откройте [http://localhost:5173](http://localhost:5173).

| Команда | Описание |
|---------|----------|
| `npm run dev` | Dev-сервер |
| `npm run build` | Production-сборка |
| `npm run preview` | Превью production-сборки |
| `npm run lint` | ESLint |
| `npm run test` / `npm run test:run` | Vitest (watch / CI) |
| `npm run build:wow-data` | Пересборка бандлов WoW JSON из `scripts/wowsims-db.json` (в т.ч. лут тира VoA из метаданных сетов, если WowSims не отдаёт зону 4603) |
| `npm run generate:bis-presets` | Пересборка встроенных BiS из `scripts/bis-list-sources.md` |
| `npm run comment:bis-presets` | Комментарии слотов в файлах BiS-пресетов |
| `npm run download:gear-slot-icons` | PNG-заглушки слотов paper-doll в `src/assets/gear-slot-icons/` |

Встроенные BiS пишутся в `scripts/bis-list-sources.md` (секции `# Class - Spec` с блоками `## Server - Author - List - URL`; гильдейские Titans — `## Titans - Guild - Titans` с русскими подписями слотов). После правок markdown перегенерируйте TypeScript-пресеты.

### Хранение

| Ключ | Содержимое |
|------|------------|
| `my-raid-cds` | Персонажи, подземелья, переключатели (`schemaVersion` 5) |
| `my-raid-cds-bis-lists` | Выбранные BiS и локальные списки (битые записи пропускаются) |
| `my-raid-cds-item-tooltip-locale` | `en` или `ru` (по умолчанию `ru`) |
| `my-raid-cds-color-mode` | Светлая/тёмная тема |

Повреждённые данные трекера сбрасываются с алертом. Старые сейвы мигрируют при загрузке.
